# SphenisOS-QEMU v3.6.4

**SphenisOS** — это легковесная 16-битная операционная система реального режима, написанная на x86 ассемблере. Проект включает в себя собственную командную оболочку, базовую файловую систему, многопользовательскую аутентификацию и интегрированные утилиты, включая полноэкранный текстовый редактор.

## Ключевые возможности

* **Оболочка Plumbum (v1.1):** Собственный командный интерпретатор с поддержкой статус-бара (отображение текущего пути и времени), обработки пакетных скриптов (`.plb`), а также выполнения логических проверок (`if`, `if_exist`).
* **Файловая система:** Базовые механизмы чтения и записи секторов, поддержка директорий, просмотр содержимого (`ls`), создание и удаление файлов/папок (`mkf`, `mkdir`, `rm`).
* **Система авторизации:** Экран входа при загрузке с разделением прав (по умолчанию `root/root`). Возможность добавления и удаления пользователей (`useradd`, `userdel`), а также смены паролей (`passwd`).
* **Текстовый редактор SNANO:** Встроенный многострочный консольный редактор для написания скриптов и заметок прямо внутри системы, дополненный утилитами для быстрого просмотра файлов (`type`).
* **Системные утилиты:** Кастомизация цвета интерфейса (`setcolor`), управление питанием (ACPI `poweroff`, `reboot`), системное время/дата (через BIOS RTC) и фирменная утилита `sysfetch` с ASCII-драконом.
* **Запуск сторонних программ:** Команда `run` позволяет загружать и исполнять внешние `.bin` файлы, размещая их в памяти по адресу `0x5000`.

## Встроенные команды

* **Навигация и ФС:** `ls`, `cd`, `pwd`, `mkdir`, `mkf`, `rm`, `fclear`
* **Текст и скрипты:** `snano`, `type`, `echo`, `write`, `plb`, `input`, `if`, `if_exist`
* **Система:** `sysfetch`, `info`, `clear`, `reboot`, `poweroff`, `time`, `date`, `setcolor`, `timeout`, `pause`
* **Пользователи:** `whoami`, `useradd`, `userdel`, `passwd`
* **Программы:** `run`

## Предустановленные приложения

Ядро содержит механизм автоматической интеграции дополнительных `.bin` файлов прямо в загрузочный образ. В текущей сборке поставляются:

* Рабочая среда и навигация: `sp_menu.bin`, `desktop.bin`
* Утилиты: `sp_calc.bin`, `hexview.bin`
* Игры и развлечения: `spspace.bin`, `sppiano.bin`

## Технические детали

* **Архитектура:** x86 (16-bit real mode)
* **Точка входа:** `0x7E00`
* **Компиляция:** Требуется компилятор NASM. Данные программ-модулей монтируются через `incbin` с выравниванием секторов (`align 512`).

## Автозапуск (autorun.plb)

Сразу после успешной авторизации в системе оболочка Plumbum автоматически ищет в корневой директории скрипт `autorun.plb`. Если файл существует, он выполняется до передачи управления пользователю. Это позволяет настроить стартовое окружение, вывести приветственные сообщения или автоматически загрузить нужную программу (например, графический интерфейс `desktop.bin`).

## Полный список команд

**Файловая система и навигация**

* `ls` - Вывод списка файлов и папок в текущей директории
* `cd <путь>` - Переход в другую директорию (поддерживает выход на уровень выше и переход в корень)
* `pwd` - Вывод текущего рабочего пути
* `mkdir <имя>` - Создание новой директории
* `mkf <имя>` - Создание нового пустого файла
* `rm <имя>` - Удаление файла
* `fclear <имя>` - Быстрая очистка содержимого существующего файла

**Работа с текстом и скриптами**

* `snano <имя>` - Запуск встроенного полноэкранного текстового редактора
* `type <имя>` - Вывод содержимого файла на экран
* `echo <текст>` - Вывод строки текста в консоль
* `write <имя> <текст>` - Прямая запись введенной строки в файл
* `plb <имя>` - Исполнение пакетного скрипта
* `input <текст>` - Запрос пользовательского ввода во время выполнения скрипта
* `if <строка>` - Логическая проверка: следующая команда выполняется только при совпадении с пользовательским вводом
* `if_exist <имя>` - Логическая проверка: следующая команда выполняется, если файл существует
* `exit` - Досрочное завершение выполнения скрипта

**Управление системой**

* `sysfetch` - Вывод информации о системе и текущем пользователе с фирменным ASCII-драконом
* `info` - Показ версий ядра ОС и оболочки
* `clear` - Очистка экрана
* `reboot` - Горячая перезагрузка системы
* `poweroff` - Выключение питания ПК (через прерывания APM)
* `time` - Просмотр текущего времени
* `date` - Просмотр текущей даты
* `setcolor <код>` - Смена цвета текста в системе (по стандартам BIOS)
* `timeout <сек>` - Остановка выполнения на заданное количество секунд
* `pause` - Приостановка работы до нажатия любой клавиши
* `help` - Вывод встроенной таблицы справки

**Пользователи и безопасность**

* `whoami` - Просмотр имени текущего авторизованного пользователя
* `useradd <имя>` - Добавление нового пользователя в систему
* `userdel <имя>` - Удаление пользователя (защита от удаления root)
* `passwd` - Смена пароля

**Исполнение программ**

* `run <имя.bin>` - Загрузка бинарного файла в память и передача ему управления

## Сборка и запуск

**Важное замечание по запуску:**
На данный момент SphenisOS спроектирована и протестирована для работы **исключительно на реальном железе**. Запуск системы на виртуальных машинах (VirtualBox, VMware) или эмуляторах (QEMU, Bochs) не гарантирует корректной работы и не поддерживается. Для тестирования необходимо записывать скомпилированный образ на физический носитель (например, USB-накопитель).

**Сборка проекта:**
В текущей версии скрипты сборки и компиляции поддерживаются **только для среды Linux**.
Для успешной компиляции ядра и всех встроенных бинарных файлов убедитесь, что у вас установлен `nasm`.

Шаги для сборки:

1. Склонируйте репозиторий.
2. Перейдите в директорию с исходным кодом.
3. Запустите скрипт сборки (`build.py`).

ВНИМАНИЕ: эта версия OS запускается только через флешку в qemu она не запуститься.
